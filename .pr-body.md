## Summary

Tree-event publication (F-015) shipped with a silo-wide `LatticeOptions.PublishEvents` boolean. This PR wraps that boolean with a per-tree override (`ILattice.SetPublishEventsEnabledAsync`) persisted on the tree's registry entry, plus a small per-activation cache (`PublishEventsGate`) that folds the override lookup into the write path without adding registry round-trips on the hot path. The silo option becomes the global default.

While wiring the gate, an existing latent bug surfaced and is fixed here: the registry tree itself (`_lattice_trees`) is backed by an `ILattice` grain whose writes flow through the same publish-gate lookup path, so without a system-tree short-circuit any write to the registry would deadlock on its own non-reentrant activation. The gate now short-circuits for trees whose id starts with `LatticeConstants.SystemTreePrefix`, honouring the silo option directly for those trees.

## Changes

### New public API

| Type | Description |
|------|-------------|
| `ILattice.SetPublishEventsEnabledAsync(bool? enabled, CancellationToken)` | Sets or clears the per-tree event-publication override. `true`/`false` force the setting; `null` removes the override so the tree inherits the silo-wide `LatticeOptions.PublishEvents` default. Persisted on `TreeRegistryEntry`. |
| `ILatticeRegistry.SetPublishEventsAsync(string treeId, bool? enabled)` | Internal registry surface backing the above. |
| `TreeRegistryEntry.PublishEvents` (`[Id(6)] bool?`) | Per-tree override field; `null` means inherit silo default. |

### Internal changes

| Component | Change |
|-----------|--------|
| `PublishEventsGate` | New per-activation gate; 5 s TTL cache, `ValueTask<bool>` on the fast path, registry-failure fallback to the silo option, `Invalidate()` for same-activation flips. **System-tree short-circuit** prevents the registry-deadlock described above. |
| 7 publishing grains | `LatticeGrain`, `AtomicWriteGrain`, `TombstoneCompactionGrain`, `TreeDeletionGrain`, `TreeReshardGrain`, `TreeResizeGrain`, `TreeShardSplitGrain`, `TreeSnapshotGrain` all gate publication through `PublishEventsGate.IsEnabledAsync(...)` instead of the prior direct `opts.PublishEvents` check. |
| `LatticeEventPublisher.PublishAsync` | Removed the internal `options.PublishEvents` re-check — callers now own gating via `PublishEventsGate`. |

### Tests (8 new)

- **4** `PublishEventsGateTests` (unit) — system-tree short-circuit honours `options.PublishEvents` (true and false) without touching `IGrainFactory`; user-tree path *does* consult the registry; null-argument guards.
- **4** `PublishEventsOverrideIntegrationTests` (integration, silo default `PublishEvents=false`) — override=true emits, override=null inherits (no events), flip true→false suppresses, clearing true→null falls back to the silo default.

### Documentation

- `docs/configuration.md` — Rewrote the `PublishEvents` section to describe it as the global default, document the resolution order (per-tree override wins; falls back to silo option), note it applies to system trees, and call out propagation semantics across silos.
- `docs/events.md` — Added a "Per-tree override" section with resolution order, propagation-delay caveat (~5 s across silos), and the system-tree exception; updated the intro to mention `SetPublishEventsEnabledAsync`.
- `docs/api.md` — Added an "Events" subsection under `ILattice` for `SetPublishEventsEnabledAsync`; added a row to the existing `LatticeExtensions` table for `SubscribeToEventsAsync` (pure additions, no content removed).

### Housekeeping

- `LatticeEventPublisher` XML doc updated to reflect that callers gate publication (the publisher no longer re-checks the silo option).
